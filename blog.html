What is the Document Object Model(DOM)?

The Document Object Model (DOM) is a programming interface for web documents. It represents the structure of a web page as a tree of objects that can be accessed and manipulated using programming languages like JavaScript. In simple terms, it is the way browsers structure the content of a webpage (HTML or XML documents) so that it can be accessed and changed dynamically.

How do you select an element from the Dom?

Selecting elements from the DOM is one of the fundamental tasks in web development. JavaScript provides several methods to retrieve elements based on their ID, class, tag, or other attributes. Use querySelector and querySelectorAll for flexibility and modern CSS selector support. Always check for null when using getElementById or querySelector to avoid runtime errors. For dynamic content, prefer querySelectorAll over getElementsBy* since it returns a static list. Use proper selectors for performance (e.g., avoid overly complex selectors).

what is event delegation in the context of the DOM, Why is it useful?

Event delegation is a technique in JavaScript where you attach a single event listener to a parent element to manage events on its child elements, even if those child elements are added dynamically after the parent has been created. Instead of adding an event listener to every child element, you use the parent element to "delegate" the event handling. This works because of event bubbling, a behavior where an event triggered on a child element propagates up the DOM tree to its ancestors.

How do you manipulate an elemets attributes and styles using the Dom?

Attributes provide additional information about an HTML element, such as id, class, href, src, etc. You can change or retrieve these attributes using the DOM. You can directly manipulate the inline CSS styles of an element via the style property of an element. This property allows you to modify any CSS property of that element directly from JavaScript.
